Обзор

CodeMirror-это компонент редактора кода, который может быть встроен в Сетевая страница. Основная библиотека предоставляет только редактор компонент, отсутствие сопровождая кнопок, автоматическ - завершения, или другого IDE функциональность. Он предоставляет богатый API, поверх которого такие функциональность может быть реализована прямолинейно. Видеть аддоны , включенные в дистрибутив, и список внешних размещенных аддонов , для многократного использования реализация дополнительных функций.

CodeMirror работает с языковыми режимами. Режимы Программы JavaScript, которые помогают цвет (и дополнительно отступ) текста написано на заданном языке. Дистрибутив поставляется с номером режимов (см. mode/ каталог), и это не трудно написать новый одни для других языков.

Основное Использование

Самый простой способ использовать CodeMirror - просто загрузить скрипт и таблица стилей lib/ , найденная в дистрибутиве, плюс скрипт режима из одного из mode/ каталогов. (См . помощник по сжатию простой способ объединить скрипты. ) Например: 

<script  src = " lib / codemirror.js ">>< / script>>
<link  rel = "stylesheet"  href ="../ lib / codemirror.css ">>
<script  src = " mode / javascript / javascript.js " >>< / script>

Сделав это, экземпляр редактора может быть создан как этот: 

var  myCodeMirror  =  CodeMirror (document .тело );

Редактор будет добавлен к тексту документа, запустится пустой, и будет использовать режим, который мы загрузили. Иметь больше контроля через новый редактор можно передать объект конфигурации к CodeMirror как второй аргумент: 

var  myCodeMirror  =  CodeMirror (document .body,  { 
  value :  "function myScript () {return 100;}\n", 
  mode :   "javascript"
});

Это инициализирует редактор с фрагментом кода уже в это, и явно сказать ему использовать режим JavaScript (который является полезно при загрузке нескольких режимов). См . ниже для полного обсуждения параметры конфигурации, которые принимает CodeMirror.

В случаях, когда вы не хотите добавлять редактор к элемент, и нужно больше контроля над путем он введен, первый аргумент CodeMirror функции также может быть функцией, которая при задании элемента DOM вставляет его в где-то документ. Это может использоваться, например, для замены textarea с реальным редактором: 

var  myCodeMirror  =  CodeMirror ( function (elt )  { 
  myTextArea .parentNode.replaceChild (elt ,  myTextArea ) ; 
} ,  { value :  myTextArea .value});

Однако для этого случая использования, который является распространенным способом использования CodeMirror, библиотека обеспечивает гораздо более мощный ярлык: 

var myCodeMirror = CodeMirror.fromTextArea (myTextArea );

Это, среди прочего, обеспечит ценность textarea обновляется с содержимым редактора, когда форма (если это часть формы) представляется. См . API ссылка для полного описания этого метода.

Конфигурация

Оба CodeMirror функция и ее fromTextArea метод принимают за вторую (необязательно) аргумент объект, содержащий параметры конфигурации. Любой вариант не поставленный как это будет принят от CodeMirror.defaults , an объект, содержащий параметры по умолчанию. Этот объект можно обновить чтобы изменить настройки по умолчанию на странице.

Опции никак не проверяются, поэтому установка фиктивной опции значения обязательно приводят к нечетным ошибкам.

Это поддерживаемые параметры:

value: string|CodeMirror.Doc
Начальное значение редактора. Может быть строкой, или объект документа .
mode: string|object
Режим для использования. Если не задано, это будет по умолчанию первый режим, который был загружен. Это может быть строка, которая либо просто называет режим или тип MIME связанный с режимом. Кроме того, это может быть объект содержащий параметры конфигурации для режима, с name свойство, называющее режим (для пример {name: "javascript", json: true} ). Демонстрация страницы для каждого режима содержат информацию о конфигурации параметры режим поддерживает. Вы можете спросить CodeMirror, какие режимы и типы MIME были определены путем проверки the CodeMirror.modes и CodeMirror.mimeModes объекты. Первые карты имена режимов для их конструкторов и вторые сопоставления типов MIME к спецификациям режима.
theme: string
Тема для оформления редактора. Вы должны убедиться, что CSS файл, определяющий соответствующий .cm-s-[name] загружаются стили (см. theme каталог в распределение.) Значение по умолчанию "default" , для которого цвета включены внутри codemirror.css . Это возможно использование сразу нескольких классов тем-для пример "foo bar" назначит оба The cm-s-foo and the cm-s-bar classes в редакцию.
indentUnit: integer
Сколько пробелов в блоке (что бы это ни значило в отредактированном язык) должны быть отступы. Значение по умолчанию-2.
smartIndent: boolean
Следует ли использовать контекстно-зависимый отступ режим предоставляет (или просто отступ такой же, как и линия раньше). По умолчанию true.
tabSize: integer
Ширина символа табуляции. По умолчанию 4.
indentWithTabs: boolean
Если, при отступе, первый N* tabSize пробелы должны быть заменены N вкладками. По умолчанию false.
electricChars: boolean
Определяет, должен ли редактор повторно отступить от текущего строка при вводе символа, который может изменить свой собственный отступ (работает, только если режим поддерживает отступ). По умолчанию true.
rtlMoveVisually: boolean
Определяет, будет ли горизонтальное перемещение курсора через справа налево (арабский, иврит) текст визуальный (нажатие левой стрелка перемещает курсор влево) или логическая (нажатие стрелки влево переходит к следующему нижнему индексу в строке, который визуально справа в тексте справа налево). Значение по умолчанию false на Windows, и true на других платформах.
keyMap: string
Настраивает карту ключей для использования. Неисполнение is "default" , который является единственным определенным сопоставлением клавиш сама по codemirror.js себе. Дополнительные карты клавиш находятся в keymap каталог. Видеть раздел на keymaps для больше информация.
extraKeys: object
Может использоваться для указания дополнительных Привязок клавиш для редактора, наряду с определенными by keyMap . Должно быть либо null, либо допустимое значение keymap.
lineWrapping: boolean
Должен ли CodeMirror прокручивать или переносить длинные строки. По умолчанию false (прокрутка).
lineNumbers: boolean
Показывать ли номера строк слева от редактора.
firstLineNumber: integer
С какого номера начать подсчет строк. По умолчанию 1.
lineNumberFormatter: function(line: integer) → string
Функция, используемая для форматирования номеров строк. Функция передал номер строки и должен вернуть строку, которая будет показано в сточной канаве.
gutters: array<string>
Смогите быть использовано для того чтобы добавить экстренные сточные канавы (за или вместо строка номер желоба). Должен быть массив имен классов CSS, каждый из которых определяет a width (и необязательно a background), и который будет использован для того чтобы нарисовать предпосылку сточная канава. Может включать CodeMirror-linenumbers класс, для того чтобы явным образом установите положение желоба номера строки (он будет по умолчанию справа от всех остальных желобов). Этот класс имена переданные ключи to setGutterMarker .
fixedGutter: boolean
Определяет, будет ли желоб прокручиваться вместе с содержимым горизонтально (false) или остается ли он фиксированным во время горизонтального прокрутка (true, по умолчанию).
coverGutterNextToScrollbar: boolean
Когда fixedGutter включен, и есть горизонтальная полоса прокрутки, по умолчанию желоб будет виден слева от этой полосы прокрутки. Если это параметр имеет значение true, он будет покрыт элементом с класс CodeMirror-gutter-filler .
readOnly: boolean|string
Это отключает редактирование содержимого редактора пользователем. Если дается специальное значение "nocursor" (вместо просто true ), фокусировка редактора также отвергнутый.
showCursorWhenSelecting: boolean
Должен ли курсор отображаться при выделении активный. По умолчанию false.
undoDepth: integer
Максимальное количество уровней отмены, сохраняемых редактором. По умолчанию 40.
historyEventDelay: integer
Период бездействия (в миллисекундах), который приведет к новое событие истории, которое будет запущено при вводе или удалении. По умолчанию 500.
tabindex: integer
Вкладка индекс , назначаемый редактору. Если не задано, нет индекса вкладки будет назначен.
autofocus: boolean
Может использоваться, чтобы CodeMirror сосредоточиться на инициализация. По умолчанию off. Когда fromTextArea это используется, и нет явного значения для этого параметра, это будет установите значение true, когда либо исходный textarea фокусируется, или это имеет autofocus атрибут и никакой другой элемент сосредоточенный.

Ниже приведены еще несколько специализированных низкоуровневых опций включенный. Эти только полезны в очень специфических ситуациях, вас может потребоваться пропустить их при первом прочтении данного руководства.

dragDrop: boolean
Определяет, включено ли перетаскивание. On по умолчанию.
onDragEvent: function(instance: CodeMirror, event: Event) → boolean
Deprecated! Посмотреть эти события обработчики для текущего рекомендуемого подхода.
Когда дано, это будет вызвано при обработке редактором a dragenter , dragover , или drop событие. Он будет передан редактору экземпляр и объект события в качестве аргументов. Обратный вызов может выберите, чтобы обработать само событие, в этом случае оно должно возврат true , чтобы указать, что CodeMirror не должен сделайте что-нибудь еще.
onKeyEvent: function(instance: CodeMirror, event: Event) → boolean

Это обеспечивает довольно низкоуровневый крюк в обработку ключа CodeMirror. Если эта функция предусмотрена, она будет вызвана каждый keydown , keyup , и keypress событие, которое CodeMirror захватывает. IT будут переданы два аргумента, экземпляр редактора и ключ событие. Это ключевое событие в значительной степени является необработанным ключевым событием, за исключением что stop() метод всегда добавляется к нему. Вы может кормить его, например, jQuery.Event далее нормализуем его.
Эта функция может проверить ключ событие, и обрабатывать его, если он хочет. Это может вернуться верно сказать CodeMirror игнорировать событие. Будьте осторожны, что, на некоторых браузерах, остановка a keydown не останавливается то keypress от стрельбы, тогда как по другим оно делает. Если вы отвечаете на событие, вам, вероятно, следует проверить свое type свойство и только делает что-то когда оно is keydown (или keypress для действий что нужны символьные данные).
cursorBlinkRate: number
Полупериод в миллисекундах, используемый для мигания курсора. Мигание по умолчанию тариф 530мс.
cursorScrollMargin: number
Сколько дополнительного пространства всегда держать над и под курсор при приближении к верхней или нижней части видимого вида в прокручиваемый документ. По умолчанию 0.
cursorHeight: number
Определяет высоту курсора. По умолчанию 1, Что означает он охватывает всю высоту линии. Для некоторых шрифтов (и по некоторые вкусы) меньшая высота (например 0.85 ), что приводит к тому, что курсор не достигает дна линии, выглядит лучше
workTime, workDelay: number
Подсветка выполняется псевдо фоновой нитью, которая будет работайте workTime миллисекунды, а затем используйте время ожидания сна в течение workDelay миллисекунд. Этот по умолчанию 200 и 300, Вы можете изменить эти параметры, чтобы сделать подсветка более или менее агрессивная.
workDelay: number
Смотрите workTime .
pollInterval: number
Указывает, как быстро CodeMirror должен опрашивать свои входные данные textarea для изменений (при фокусировке). Большинство входных данных захватывается события, но некоторые вещи, такие как IME input в некоторых браузерах, не генерируйте события, которые позволяют CodeMirror правильно его обнаружить. Таким образом, это опросы. По умолчанию 100 миллисекунд.
flattenSpans: boolean
По умолчанию CodeMirror объединит смежные маркеры в одиночный промежуток, если они имеют один и тот же класс. Это приведет к проще дерево DOM, и, таким образом, выполнять лучше. С некоторыми видами укладка (например, закругленные углы), это изменит способ выглядит документ. Чтобы отключить этот параметр, можно установить значение false поведение.
maxHighlightLength: number
При выделении длинных линий, для того, чтобы оставаться отзывчивым, редактор откажется и просто оформит остальную часть строки как обычный текст, когда он достигает определенной позиции. Значение по умолчанию 10000. Вы можете установить это Infinity значение, чтобы выключить такое поведение.
viewportMargin: integer
Указывает количество строк, отображаемых выше, и ниже части документа, которая в настоящее время прокручивается в вид. Это влияет на количество обновлений, необходимых при прокрутке, и объем работы, которую выполняет такое обновление. Вы должны обычно оставьте его по умолчанию, 10. Может устанавливать Infinity чтобы убедиться, что весь документ всегда визуализируется, и, таким образом, текстовый поиск браузера работает на нем. Это будет иметь плохие влияния на представлении большого документы.

Мероприятия

Различные связанные с CodeMirror объекты излучают события, которые позволяют клиентский код для реагирования на различные ситуации. Обработчики для таких события могут быть зарегистрированы on и off методы на объектах что событие срабатывает. Для запуска собственных событий, использовать CodeMirror.signal(target, name, args...) , где target находится объект non-DOM-node.

Экземпляр редактора запускает следующие события. instance Аргумент всегда ссылается на редактор себя.

"change" (instance: CodeMirror, changeObj: object)
Срабатывает при каждом изменении содержимого редактора. changeObj Объект, {from, to, text, removed, next} содержащий информацию об изменениях это было вторым аргументом. from и to позиции (в пре-изменении система координат), где началось и закончилось изменение (для пример, он мог быть {ch:0, line:18} если позиция в начале строки №19). text is массив строк, представляющих текст, который заменил измененный диапазон (разделение по линии). removed есть текст что раньше было между from и to , который перезаписан этим изменением. Если несколько изменений произошло во время одной операции, объект будет иметь next свойство, указывающее на другой объект изменения (что может указывать на другое и т.д.).
"beforeChange" (instance: CodeMirror, changeObj: object)
Это событие запускается перед применением изменения и его обработчик может изменить или отменить изменение. Объект changeObj имеет from , to , и text свойства, как с "change" событие, но never a next property, since this is fired for each индивидуальное изменение, и не дозировано в деятельность. Он также имеет cancel() метод, который может быть вызван для отмены изменение, и, если изменение не приходит из события undo или redo update(from, to, text) метод, который может быть использован для изменения изменения. Отменить или повторить изменения невозможно, так как они содержат некоторые изменения метаинформация для восстановления только старых отмеченных диапазонов действительно для этого конкретного изменения. Все три аргумента к update опционны, и могут быть выйдены к оставьте существующее значение для этого поля нетронутый. Примечание: Вы не можете сделать ничего от есть "beforeChange" обработчик, который вызовет изменения к документу или его визуализации. Делать так будет, так как это обработчик вызывается непосредственно из недр CodeMirror реализация, вероятно, приведет к тому, что редактор станет испорченный.
"cursorActivity" (instance: CodeMirror)
Будет запущен при перемещении курсора или выделенного элемента, или изменяется содержимое редактора.
"keyHandled" (instance: CodeMirror, name: string, event: Event)
Увольняется после обработки ключа через раскладка. name имя обрабатываемого ключа (для пример "Ctrl-X" или "'q'" ), and event is the DOM keydown или keypress событие.
"inputRead" (instance: CodeMirror, changeObj: object)
Срабатывает всякий раз, когда новый ввод считывается из скрытого textarea (введено или вставлено пользователем).
"beforeSelectionChange" (instance: CodeMirror, selection: {head, anchor})
Это событие запускается перед перемещением выборки. Свой обработчик может изменять результирующую головку и якорь выборки. selection Параметр является объектом С head и anchor свойства хранение {line, ch} объектов, которые обработчик может чтение и обновление. Обработчики для этого события имеют то же самое ограничение как "beforeChange" обработчики-они не должны делать ничего, чтобы непосредственно обновить состояние редактора.
"viewportChange" (instance: CodeMirror, from: number, to: number)
Срабатывает всякий раз, когда порт представления редактор изменяется (из-за прокрутки, редактирования или любого другого фактор.) То from и to аргументы задайте новое начало и конец окна просмотра.
"gutterClick" (instance: CodeMirror, line: integer, gutter: string, clickEvent: Event)
Срабатывает, когда желоб редактора (область номер строки) нажатый. Передаст экземпляр редактора в качестве первого аргумента, (на основе нуля) номер строки, которая была нажата в качестве второй аргумент, CSS-класс желоба, который был нажат как третий аргумент и необработанный mousedown объект события как четвертый аргумент.
"focus" (instance: CodeMirror)
Срабатывает при фокусировке редактора.
"blur" (instance: CodeMirror)
Срабатывает при расфокусировке редактора.
"scroll" (instance: CodeMirror)
Срабатывает при прокрутке редактора.
"update" (instance: CodeMirror)
Будет уволен всякий раз, когда CodeMirror обновляет свой дисплей DOM.
"renderLine" (instance: CodeMirror, line: LineHandle, element: Element)
Срабатывает всякий раз, когда линия (повторно)отображается DOM. Обожженный сразу после построения элемента DOM добавлено в документ. Обработчик может испортить стиль результирующий элемент или добавить обработчики событий, но не следует пытаться изменить состояние редактора.
"mousedown", "dblclick", "contextmenu", "keydown", "keypress", "keyup", "dragstart", "dragenter", "dragover", "drop" (instance: CodeMirror, event: Event)
Срабатывает, когда CodeMirror обрабатывает событие DOM этого типа. Вы можете preventDefault событие, или дать ему codemirrorIgnore свойство truthy, чтобы сигнализировать, что CodeMirror не должен выполнять дальнейшую обработку.

Объекты документа (экземпляры of CodeMirror.Doc ) излучают последующие события:

"change" (doc: CodeMirror.Doc, changeObj: object)
Увольняется всякий раз, когда происходит изменение документ. changeObj имеет подобный тип как объект, переданный редактор "change" событие, но оно никогда не имеет next свойства, потому что события изменения документа не пакетные (в то время как изменение редактора события).
"beforeChange" (doc: CodeMirror.Doc, change: object)
См . Описание то же самое событие в экземплярах редактора.
"cursorActivity" (doc: CodeMirror.Doc)
Срабатывает всякий раз, когда курсор или выделение в этом документе изменения.
"beforeSelectionChange" (doc: CodeMirror.Doc, selection: {head, anchor})
Эквивалентный событие то же самое имя как запущено в экземплярах редактора.

Линейные маркеры (как возвращено, для пример, getLineHandle ) поддержка этих событий:

"delete" ()
Будет запущен при удалении объекта line. Линейный объект связан с началом строки. В основном полезно когда вам нужно узнать, когда ваш желоб маркеры на заданной линии удаляются.
"change" (line: LineHandle, changeObj: object)
Срабатывает при любом изменении текстового содержимого строки (но строка не удаляется сразу). The change объект похож на переданный изменение события в Редакторе объект.

Маркированные маркеры диапазона ( CodeMirror.TextMarker ), как возвращено by markText и setBookmark , излучают последующие события:

"beforeCursorEnter" ()
Срабатывает, когда курсор входит в отмеченный диапазон. Из этого обработчик событий, состояние редактора может быть проверено но не изменен, за исключением того, что диапазон на которые события пожаров могут быть очищены.
"clear" (from: {line, ch}, to: {line, ch})
Срабатывает, когда диапазон очищен, либо через курсор движение в комбинации с clearOnEnter или через вызов его clear() метода. Будет только быть уволенным раз в ручку. Обратите внимание, что удаление диапазона через редактирование текста не запускает это событие, поскольку действие отмены может вернуть диапазон в существование. from и to дайте часть документа, что диапазон spanned, когда он был очищен.
"hide" ()
Срабатывает при удалении последней части маркера из операции редактирования документа.
"unhide" ()
Срабатывает, когда после удаления маркера путем редактирования выполняется отмена операция вернула маркер.

Виджеты линии ( CodeMirror.LineWidget ), возвращено by addLineWidget , огонь такие случаи:

"redraw" ()
Запускается при каждом повторном добавлении виджета редактором в DOM. Это произойдет один раз сразу после добавления виджета (если это прокручивается в поле зрения), а затем снова всякий раз, когда он прокручивается или при изменении параметров редактора или линия, на которой находится виджет, требует, чтобы виджет был перерисованный.

Раскладки

Карты клавиш-это способы связывания Ключей с функциональностью. Раскладка является строками сопоставления объектов, определяющими ключи к функциям которые реализуют свои функциональные возможности.

Ключи идентифицируются по имени или символу. CodeMirror.keyNames Объект определяет имена для общие ключи и связывает их с их ключевыми кодами. Примеры имена, определенные здесь Enter , F5 , и Q . Эти можно приставить с Shift- , Cmd- , Ctrl- , и Alt- (в таком порядке! ) для указания модификатора. Так например, Shift-Ctrl-Space будет действительным ключом идентификатор.

Общий пример: сопоставьте клавишу Tab для вставки пробелов вместо вкладки характер. 

{ 
  Tab :  function ( cm )  { 
    var  spaces  =  Array (cm .getOption ("indentUnit")  +  1 ) .join ( "" ); 
    cm .replaceSelection (пробелы,  "end", " +input");
  }
}

Кроме того, символ может быть указан непосредственно окружение его в одинарных кавычках, например '$' или 'q' . Из-за ограничений в способе запуска браузеров ключевые события, они не могут иметь префиксов с модификаторами.

CodeMirror.keyMap Объект связывает keymaps с названиями. Определения кода пользователя и карты ключей можно назначить дополнительно свойства этого объекта. В любом месте, где ожидается можно задать строку, которая будет просматриваться в этом объекте. IT также содержит "default" keymap держа привязка по умолчанию.

Значения свойств в ключевых картах могут быть один аргумент (экземпляр CodeMirror), строки или false . Такие строки относятся к свойствам CodeMirror.commands объект, определяющий число общие команды, используемые привязками клавиш по умолчанию и картами им к функциям. Если свойство имеет значение false , CodeMirror оставляет обработку ключа до браузера. Ключ функция обработчика может возвратиться CodeMirror.Pass для указания что он решил не обращаться с ключом, а с другими обработчиками (или поведение по умолчанию) должен быть задан поворот.

Ключи, сопоставленные с именами команд, которые начинаются с символы "go" (которые должны использоваться для действия перемещения курсора) будет запущен, даже если дополнительный Shift модификатор присутствует (т. е. "Up": "goLineUp" соответствует и вверх и сдвиг вверх). Это используется для легко реализовать shift-selection.

Keymaps можно передавать друг другу путем определения fallthrough свойство. Это указывает на то, что при ключ не найден в самой карте, одна или несколько других карт должны поиск. Оно может держать или одиночное keymap или блок раскладки.

Когда keymap содержит nofallthrough свойство установите в true , ключи соответствуют этой карте будет игнорируется, если они не соответствуют каким-либо привязкам на карте (нет дальнейшие дочерние карты будут опробованы). Когда disableInput свойство установлено to true , эффект вставки символа по умолчанию будет подавлен, когда карта клавиш активна как верхний уровень карта.

До определенной степени, взгляд CodeMirror может быть изменен мимо изменение файлов таблицы стилей. Таблицы стилей, поставляемые режимами просто обеспечьте цветы для того режима, и смогите быть приспособлено в a очень простой способ. Некоторая осторожность должна быть там, так как много правил в этом файл необходим для правильной работы функции CodeMirror. Установочный цветы должны быть безопасны, конечно, и с некоторой заботой много другие вещи также могут быть изменены. Классы CSS, определенные в этот файл выполняет следующие роли:

CodeMirror
Внешний элемент редактора. Это должно быть использовано для редактор ширины, высоты, границ и позиционирования. Смогите также быть использовано чтобы задать стили, которые должны храниться для всего в Редакторе (например, шрифт и размер шрифта), или установить фон.
CodeMirror-scroll
Прокручивается ли редактор ( overflow: auto + фиксированная высота.) По умолчанию это так. Установочный CodeMirror класс, чтобы иметь height: auto и давая этот класс overflow-x: auto; overflow-y: hidden; вызовет редактор чтобы изменить размер, чтобы соответствовать его содержание .
CodeMirror-focused
Всякий раз, когда редактор фокусируется, верхний элемент получает это класс. Используется для скрытия курсора и выделения другой цвет, когда редактор не сфокусирован.
CodeMirror-gutters
Это фон для всех желобов. Используйте его для установки цвет фона желоба по умолчанию и при необходимости добавление границы справа от желобов.
CodeMirror-linenumbers
Используйте это для придания фона или ширины номеру линии водосточная канава.
CodeMirror-linenumber
Используется для стилизации фактических номеров отдельных строк. Эти не будет детей CodeMirror-linenumbers (множественное число) элемент, но скорее будет абсолютно расположен к наложить его. Используйте это, чтобы задать выравнивание и свойства текста для номер строки.
CodeMirror-lines
Видимых строк. Здесь указывается вертикаль заполнение содержимого редактора.
CodeMirror-cursor
Курсор-это абсолютно расположенный элемент блока. Вы можете сделать его выглядеть так, как вы хотите.
CodeMirror-selected
Выделение представлено span элементами с этим классом.

Они используются для стилизации соответствующих (или несопоставленных) скобок.

API программирования

Многие функции CodeMirror доступны только через API-ИНТЕРФЕЙС. Таким образом, вам нужно написать код (или используйте аддоны ), если вы хотите предоставить им ваш пользователь.

При представлении точек в документе API использует объекты со line ch свойствами. Оба основаны на нуле. CodeMirror убеждается "закрепить" все положения передается клиентским кодом так, чтобы они помещались внутри документа, поэтому вы не стоит слишком беспокоиться о очистке ваших координат. Если вы дайте ch значение null , или не укажите его, он будет заменен на длину указанной линия.

Методы с префиксом doc. can, если не иначе задано, вызывается оба на CodeMirror (редактор) экземпляры и CodeMirror.Doc экземпляры. Методы с префиксом cm. доступны только на CodeMirror экземплярах.

Конструктор

Построение экземпляра редактора выполняется с помощью CodeMirror(place: Element|fn(Element), ?option: object) конструктор. Если place аргумент является элементом DOM, к нему будет добавлен редактор. Если это функция, она будет вызвана и, как ожидается, разместит редактор документа. options может быть элементом сопоставление имен параметров со значениями. Опцион что он явно не указывает (или все параметры, если это не так пройдено) будет принято от CodeMirror.defaults .

Обратите внимание, что объект options, переданный конструктору, будет мутировал, когда параметры экземпляра меняются , поэтому не стоит делиться такими объекты между экземплярами.

Посмотреть CodeMirror.fromTextArea для другого способа создания экземпляра редактора.

Отключить добавленную карту клавиш С. addKeyMap Любой передайте сам объект keymap или строку, которая будет сравненный против name свойства активного раскладки.
cm.addOverlay(mode: string|object, ?options: object)
Включите выделение наложения. Это мини-режим без состояния это может быть использовано для добавления дополнительной подсветки. Например, аддон поиска использует его для выделите термин, который в настоящее время используется разыскиваемый. mode может быть режим спецификация или объект mode (объект с a token method). options Параметр является необязательным. Если дано, то оно должен быть объект. В настоящее время, только opaque опция распознана. Это значение по умолчанию off, но может быть дано разрешить стиль наложения, когда нет null , к полностью переопределите стиль базового режима вместо два применяется вместе.
cm.removeOverlay(mode: string|object)
Передайте это точное значение переданное для mode параметр к addOverlay , или строка, соответствующая свойству name это значение, чтобы снова удалить наложение.
cm.on(type: string, func: (...args))
Регистрация обработчика событий для данного типа событий (a string) на экземпляре редактора. Есть также версия CodeMirror.on(object, type, func) это позволяет регистрировать события на любом объекте.
cm.off(type: string, func: (...args))
Удалите обработчик событий на экземпляре редактора. - эквивалент CodeMirror.off(object, type, func) также существует.

Методы управления документами

Каждый редактор связан с экземпляром конечно CodeMirror.Doc , свой документ. Документ представляет содержимое редактора, а также выделение и историю отмены, и режим . Документ может быть только связано с одним редактором одновременно. Вы можете создать новый документы путем вызова CodeMirror.Doc(text, mode, firstLineNumber) конструктора. Последние два аргумента необязательно и может использоваться для установки режима для документа и он начинается с номера строки, отличного от 0, соответственно.

cm.getDoc() → Doc
Извлеките текущий активный документ из редактора.
doc.getEditor() → CodeMirror
Получение редактора, связанного с документом. Может возвращение null .
cm.swapDoc(doc: CodeMirror.Doc) → Doc
Прикрепите новый документ к редактору. Возвращает старый документ, который больше не связан с редактором.
doc.copy(copyHistory: boolean) → Doc
Создайте идентичную копию данного документа. Когда copyHistory это правда, история также будет скопированный. Не может быть вызван непосредственно на Редакторе.
doc.linkedDoc(options: object) → Doc
Создайте новый документ, связанный с целевым документом. Связанные документы будут синхронизироваться (изменения в один также применяется к другим) пока не разорвется . Эти параметры поддерживаются:
sharedHist: boolean
При включении связанная копия будет совместно использовать функцию отмены история с оригиналом. Таким образом, что-то сделано в одном из два могут быть отменены в другом, и наоборот.
from: integer
to: integer
Может быть дано, чтобы сделать новый документ оригинальный. Подвиды показывают только заданный диапазон линий. Отмечать координаты этой линии внутри подвида будут согласованными с теми из родителя, так что, например, subview начиная с строки 10, первая строка будет называться строкой 10, не 0.
mode: string|object
По умолчанию новый документ наследует режим родитель. Этот параметр можно установить в спецификация режима , чтобы дать ему другой режим.
doc.unlinkDoc(doc: CodeMirror.Doc)
Разорвать связь между двумя документами. После вызова этого, изменения больше не будут распространяться между документами, и, если у них была общая история, история станет отделять.
doc.iterLinkedDocs(function: (doc: CodeMirror.Doc, sharedHist: boolean))
Вызовет данную функцию для всех документов, связанных с целевой документ. Будет передано два аргумента, связанный документ и логическое значение, указывающее, имеет ли этот документ общую историю с мишенью.

Методы, связанные с историей

doc.undo()
Отменить одно изменение (если сохраняются события отмены).
doc.redo()
Redo one undone edit.
doc.historySize() → {undo: integer, redo: integer}
Возвращает объект со {undo, redo} свойствами, оба из которых содержат целые числа, указывающие количество сохраненных операции отмены и повтора.
doc.clearHistory()
Очищает историю отмены редактора.
doc.getHistory() → object
Получить (JSON-сериализуемое)представление истории отмены.
doc.setHistory(history: object)
Замените историю отмены редактора на предоставленную, которое должно быть возвращенным значением by getHistory . Заметить что это будет иметь полностью неопределенные результаты, если содержимое редактора не так же, как это было, когда getHistory был называемый.

Методы маркировки текста

doc.markText(from: {line, ch}, to: {line, ch}, ?options: object) → TextMarker
Может использоваться для обозначения диапазона текста с определенным CSS имя класса. from и to должен будьте {line, ch} объектами. The options параметр является необязательным. Когда дано, это должен быть объект, который может содержать следующие параметры конфигурации:
className: string
Присваивает класс CSS отмеченному фрагменту текста.
inclusiveLeft: boolean
Определяет ли текст, вставленный слева от маркера, окажется внутри или вне его.
inclusiveRight: boolean
Как inclusiveLeft , но для правой стороны.
atomic: boolean
Атомарные диапазоны действуют как единое целое при движении курсора обеспокоенный-т. е. невозможно поместить курсор внутрь их. В атомных диапазонах, inclusiveLeft и inclusiveRight имеют другое значение-они предотвращает размещение курсора соответственно сразу перед и сразу после рядом.
collapsed: boolean
Свернутые диапазоны не отображаются на дисплее. Установка a диапазон, который будет свернут, автоматически сделает его атомарным.
clearOnEnter: boolean
При включении, заставит метку очистить себя всякий раз, когда курсор входит в свой диапазон. Это в основном полезно для виджеты замены текста, которые должны "щелкнуть", когда пользователь пытается изменить их. Этот "clear" событие увольнянный на ручке ряда можно использовать для того чтобы быть сообщенным когда это происходит.
replacedWith: Element
Используйте данный узел для отображения этого диапазона. Подразумевает оба коллапсировал и атомный. Данный узел DOM должен быть встроенный элемент (в отличие от блочного элемента).
handleMouseEvents: boolean
Когда replacedWith задано, это определяет будет ли редактор захватывать события мыши и перетаскивания происходящие в этом виджете. По умолчанию false-события будут left alone for the default browser handler, or specific обработчики на виджете, для захвата.
readOnly: boolean
Только для чтения промежуток может, пока он не очищен, не быть изменено, за исключением вызов setValue для сброса весь документ. Примечание: добавление диапазона только для чтения в настоящее время очищает журнал отмены редактора, потому что существующие события отмены частично аннулируются только для чтения пяди повредили бы историю (в текущем реализация.)
addToHistory: boolean
Если установлено значение true (по умолчанию false), добавление этого маркера создаст событие в истории отмены, которое может индивидуально отменить (очистка маркера).
startStyle: string
Может использоваться для указания дополнительный класс CSS, который будет применен к самому левому промежутку является частью маркера.
endStyle: string
Аналог К startStyle , но для самого правого пролета.
title: string
Когда задано, даст созданные узлы для этого промежутка title атрибут HTML с заданное значение.
shared: boolean
Когда целевой документ связан с другими документы, вы можете установить shared к true для того чтобы сделать маркер появляется во всех документах. По умолчанию отображается маркер только в целевом документе.
Метод возвращает объект, представляющий маркер (с constuctor CodeMirror.TextMarker ), который предоставляет три метода: clear() , удалить метку, find() , который возвращается {from, to} объект (оба документа позиции), указывающие текущее положение отмеченного диапазона, или undefined если маркера больше нет в документ, и наконец changed() , который вы можете позвонить, если вы сделали что-то, что может измениться размер маркера (например изменение содержимого a replacedWith node), и хотите дешево обновить дисплей.
doc.setBookmark(pos: {line, ch}, ?options: object) → TextMarker
Вставка закладки, маркера, который следует за текстом вокруг него как это редактируется, на данной позиции. Закладка имеет два методы find() И. clear() Первый возвращает текущее положение закладки, если она все еще находится в документ, а второй явно удаляет закладку. Аргумент options является необязательным. Если дано, следующее свойства распознаются:
widget: Element
Может использоваться для отображения DOM узел в текущем местоположении закладки (аналогично the replacedWith вариант к markText ).
insertLeft: boolean
По умолчанию набран текст при наведении курсора на верхнюю закладку справа от закладки. Установите для этого параметра значение true вместо этого налево.
doc.findMarksAt(pos: {line, ch}) → array<TextMarker>
Возвращает массив всех закладок и отмеченных диапазонов присутствует на данной должности.
doc.getAllMarks() → array<TextMarker>
Возвращает массив, содержащий все отмеченные диапазоны в документе.

Виджет, желоб и методы оформления

cm.setGutterMarker(line: integer|LineHandle, gutterID: string, value: Element) → LineHandle
Задает маркер желоба для данного желоба (определяется его класс CSS, см. опция gutters ) к заданному значению. Значение может быть либо null , чтобы очистите маркер или элемент DOM, чтобы установить его. элемент DOM будет отображаться в указанном желобе рядом с указанным линия.
cm.clearGutter(gutterID: string)
Удалить все маркеры желоба в желоб с заданным идентификатором.
cm.addLineClass(line: integer|LineHandle, where: string, class: string) → LineHandle
Задайте имя класса CSS для данной строки. line может быть номером или линейным дескриптором. where определяет к какому элементу должен применяться этот класс, может быть один of "text" (текстовый элемент, который лежит перед выделение), "background" (элемент фона что будет за выбором), или "wrap" (the узел обертки, который обертывает все элементы линии, включая элементы желоба). class должно быть имя класс для применения.
cm.removeLineClass(line: integer|LineHandle, where: string, class: string) → LineHandle
Удалите класс CSS из строки. line может быть линия ручка или номер. where должен быть один of "text" , "background" , или "wrap" (см. addLineClass ). class можно отключить, чтобы удалить все классы для указанного узла, или быть строкой для удаления только определенного класса.
cm.lineInfo(line: integer|LineHandle) → object
Возвращает номер строки, текстовое содержимое и состояние маркера данная строка, которая может быть либо номером, либо дескриптором строки. Возвращаемый объект имеет структуру {line, handle, text, gutterMarkers, textClass, bgClass, wrapClass, widgets} , где gutterMarkers находится идентификатор желоба отображения объекта к элементам маркера и widgets является массивом линейные виджеты , прикрепленные к этому линии и различные свойства класса относятся к добавленным классам С. addLineClass
cm.addWidget(pos: {line, ch}, node: Element, scrollIntoView: boolean)
Ставит node , который должен быть абсолютно расположенный узел DOM в Редакторе, расположенный прямо под данная {line, ch} позиция. Если scrollIntoView значение true, редактор обеспечит что весь узел виден (если это возможно). Чтобы удалить виджет снова, просто использовать методы DOM (переместить его в другое место, или звоните removeChild своему родителю).
cm.addLineWidget(line: integer|LineHandle, node: Element, ?options: object) → LineWidget
Добавляет виджет линии, элемент, показанный под линией, охватывающий вся ширина редактора и перемещение строк под ним вниз. line должно быть целым числом или a линия дескриптор, и node должен быть узлом DOM, который будет отображаться под данной строкой. options , если задано, должен быть объект, который настраивает поведение штучка. Поддерживаются следующие параметры (по умолчанию ложный):
coverGutter: boolean
Должен ли виджет закрывать желоб.
noHScroll: boolean
Должен ли виджет оставаться фиксированным горизонтальная прокрутка.
above: boolean
Заставляет виджет быть помещен выше, а не ниже текст строки.
showIfHidden: boolean
При значении true виджет будет отображаться, даже если линия, с которой она связана, скрыта.
handleMouseEvents: boolean
Определяет, будет ли редактор захватывать мышь и перетащите события, происходящие в этом виджете. По умолчанию false— события будут оставлены в покое для обработчика браузера по умолчанию, или конкретные обработчики на виджете, чтобы захватить.
insertAt: integer
По умолчанию виджет добавляется под другими виджетами для линия. Этот параметр можно использовать для размещения в другом месте положение (нуль для верхней части, N для того чтобы положить его после nth другого штучка.) Заметьте что это только имеет влияние раз, когда виджет создан.
Обратите внимание, что узел виджета станет потомком узлов с CodeMirror-определенные CSS классы, и эти классы могут в некоторых случаи влияют на него. Этот метод возвращает объект, представляющий размещение виджета. Это будет иметь line свойство указывая на линейный маркер, с которым он связан, и следующие методы:
clear()
Удаляет виджет.
changed()
Вызов это если вы внесли некоторые изменения в узел DOM виджета, который может повлиять на его высоту. Это заставит CodeMirror обновить высота строки, содержащей виджет.

Методы определения размеров, прокрутки и позиционирования

cm.setSize(width: number|string, height: number|string)
Программно задайте размер редактора (переопределяя применимые CSS правила ). width и height высота может быть либо числа (интерпретируются как пиксели) или CSS единиц ( "100%" например). Вы можете пройдите null для любого из них, чтобы указать, что размер не должен быть изменен.
cm.scrollTo(x: number, y: number)
Прокрутите редактор до заданной (пиксельной) позиции. Оба аргументы могут быть оставлены как null или undefined не иметь никакого эффекта.
cm.getScrollInfo() → {left, top, width, height, clientWidth, clientHeight}
Получение {left, top, width, height, clientWidth, clientHeight} объекта, представляющего текущую прокрутку положение, размер прокручиваемой области и размер видимая область (минус полосы прокрутки).
cm.scrollIntoView(pos: {line, ch}|{left, top, right, bottom}, ?margin: number)
Прокручивает данный элемент в поле зрения. pos может быть либо {line, ch} позиция, относящаяся к заданному символ, null , для ссылки на курсор, или {left, top, right, bottom} объект, в редактор-локальные координаты. Параметр margin необязательный. Когда задано, это указывает на количество пикселей вокруг данная область, которая также должна быть видимой.
cm.cursorCoords(where: boolean|{line, ch}, mode: string) → {left, top, bottom}
Возвращает объект {left, top, bottom} содержит координаты позиции курсора. Если mode есть "local" , то они будут относительно левого верхнего угла редактируемого документа. Если это дано "page" или не дано, они относительно верхний левый угол страницы. where может быть логическим указывает, хотите ли вы, чтобы start ( true ) или end ( false ) выделения, или, если {line, ch} объект задан, он указывает точное положение на который вы хотите измерить.
cm.charCoords(pos: {line, ch}, ?mode: string) → {left, right, top, bottom}
Возвращает позицию и размеры произвольного характер. pos должно быть a {line, ch} объект. Это отличается от cursorCoords того, что это даст размер всего персонажа, а не только положение, которое курсор будет иметь, когда он будет сидеть на что позиция.
cm.coordsChar(object: {left, top}, ?mode: string) → {line, ch}
Данный {left, top} объект возвращает {line, ch} положение, которое ему соответствует. Этот необязательный mode параметр определяет относительно чего координаты интерпретируются. Это может be "window" , "page" (по умолчанию), или "local" .
cm.lineAtHeight(height: number, ?mode: string) → number
Вычисляет линию в заданном пикселе высота. mode может быть одной из тех же строк что coordsChar примет.
cm.heightAtLine(line: number, ?mode: string) → number
Вычисляет высоту вершины линии в координате система, указанная mode (см. coordsChar ), которая по умолчанию "page" . Когда линия под дном документ указан, возвращаемое значение находится внизу последняя строка в документе.
cm.defaultTextHeight() → number
Возвращает высоту строки шрифта по умолчанию для редактора.
cm.defaultCharWidth() → number
Возвращает ширину пикселя ' x ' в шрифте по умолчанию для редактор. (Обратите внимание, что для шрифтов не-monospace это в основном бесполезно, и даже для шрифтов monospace, не ascii символов может иметь другую ширину).
cm.getViewport() → {from: number, to: number}
Возвращает {from, to} объект, указывающий начало (включительно) и конец (исключительно) текущего отображения часть документа. В больших документах, когда большинство содержимого прокручивается вне поля зрения, CodeMirror будет отображать только видимое часть и поле вокруг нее. Смотреть также the viewportChange событие.
cm.refresh()
Если ваш код что-то делает, чтобы изменить размер редактора элемент (изменения размера окна уже прослушиваются) или отображение это, вы, вероятно, должны следовать, вызывая этот метод, чтобы убедитесь, что CodeMirror по-прежнему выглядит так, как задумано.

Методы, связанные с режимом, состоянием и маркерами

При написании функциональности, учитывающей язык, она часто может быть полезно зацепиться за знание того, что язык CodeMirror режим имеет. См . раздел о режимах для a более подробное описание того, как они работают.

doc.getMode() → object
Возвращает объект (внешний) mode для редактора. Обратите внимание, что это отличается от getOption("mode") , который дает вам спецификация режима, а не разрешенный экземпляр mode object .
doc.getModeAt(pos: {line, ch}) → object
Возвращает внутренний режим в заданном положении. Это вернется то же getMode , что и для простые режимы, но вернет внутренний режим для вложенности режимов (например htmlmixed ).
cm.getTokenAt(pos: {line, ch}, ?precise: boolean) → object
Получает сведения о токене, найденном в текущем режиме перед заданной позицией ( {line, ch} объектом). Этот возвращаемый объект имеет следующие свойства:
start
Символ (на данной строке), с которого начинается токен.
end
Символ, на котором заканчивается токен.
string
Строка токена.
type
Тип маркера назначенный режим к токену, например "keyword" или "comment" (также может быть null).
state
Состояние режима в конце этого маркера.
Если precise значение true, токен будет гарантированно точным на основе последних изменений. Если false или не указано, маркер будет использовать кэшированную информацию о состоянии, которая будет быстрее, но может оказаться неточной, если изменения были сделаны недавно и выделение еще не завершено.
cm.getTokenTypeAt(pos: {line, ch}) → string
Это (намного) более дешевая версия getTokenAt полезно для когда вам просто нужен тип токена в заданной позиции, и никакой другой информации. Вернется null за неструктурированные маркеры и строка, потенциально содержащая несколько имена стилей, разделенные пробелами, в противном случае.
cm.getHelper(pos: {line, ch}, type: string) → helper
Получить соответствующий помощник для данной позиции. Прислуга укажите способ поиска функциональных возможностей, подходящих для режима. type Аргумент предоставляет вспомогательное пространство имен (видеть также registerHelper ), в котором будет найдено значение. Ключ, который используется зависит от режима , активного в данный момент позиция. Если объект mode содержит свойство с тем же имя в качестве type аргумента, который используется первым. Затем, режим helperType , если таковой имеется, пробуется. И наконец, имя режима.
cm.getStateAfter(?line: integer, ?precise: boolean) → object
Возвращает состояние синтаксического анализатора режима, если таковое имеется, в конце заданный номер строки. Если номер строки не указан, состояние на возвращается конец документа. Это может быть полезно для хранения разбор ошибок в состоянии или получение других видов контекстная информация для строки. precise определяется как в getTokenAt() .

Разные методы

cm.operation(func: () → any) → any
CodeMirror внутренне буферизует изменения и только обновляет его Структура DOM после того как она заканчивала выполнить некоторую деятельность. Если вам нужно выполнить много операций над CodeMirror экземпляр, можно вызвать этот метод с аргументом функции. IT вызовет функцию, буферизуя все изменения, и только делая дорогостоящее обновление после возврата функции. Это может быть намного быстрее. Возвращаемое значение этого метода будет возвращаемым значение вашей функции.
cm.indentLine(line: integer, ?dir: string|integer)
Отрегулируйте отступ данной линии. Второй аргумент (по умолчанию "smart" ) может быть одним из:
"prev"
Базовый отступ на отступ предыдущей строки.
"smart"
Используйте умный отступ режима если доступный, ведите себя как "prev" иначе.
"add"
Увеличьте отступ линии мимо один блок отступа .
"subtract"
Уменьшите отступ линии.
<integer>
Добавить (положительное число) или уменьшить (отрицательное число) отступ по заданному количеству пробелов.
cm.toggleOverwrite(?value: bool)
Переключение между режимом перезаписи и обычным режимом вставки (если нет данный аргумент), или устанавливает режим перезаписи на определенный состояние (когда задан аргумент).
doc.posFromIndex(index: integer) → {line, ch}
Вычисляет и возвращает {line, ch} объект для a index значение ВОЗ на основе нуля относительно начала текст редактора. Если index находится вне диапазона текста, то возвращаемый объект обрезается до начала или конца текста соответственно.
doc.indexFromPos(object: {line, ch}) → integer
Обратная сторона posFromIndex .
cm.focus()
Дайте редактору фокус.
cm.getInputField() → TextAreaElement
Возвращает скрытый textarea, используемый для чтения ввода.
cm.getWrapperElement() → Element
Возвращает узел DOM, представляющий редактор, и контролирует его размер. Удалите это из дерева, чтобы удалить экземпляр редактора.
cm.getScrollerElement() → Element
Возвращает узел DOM, ответственный за прокрутку редактора.
cm.getGutterElement() → Element
Извлекает узел DOM, содержащий желоба редактора.

Статическое свойство

Сам CodeMirror объект предоставляет несколько полезных свойств.

CodeMirror.version: string
Он содержит строку, которая указывает версию библиотека. Это тройка целые числа "major.minor.patch" , где patch ноль для релизов, а что-то else(обычно один) для снимков dev.
CodeMirror.fromTextArea(textArea: TextAreaElement, ?config: object)
Метод предоставляет другой способ инициализации редактора. Он принимает a узел textarea DOM в качестве первого аргумента и необязательной конфигурации объект как второй. Он заменит textarea на CodeMirror экземпляр, и провод вверх по форме того textarea (если любой), то для того чтобы сделать убедитесь, что содержимое редактора помещено в textarea, когда форма представляемый. Экземпляр CodeMirror, созданный таким образом, имеет три дополнительный метод:
cm.save()
Скопируйте содержимое редактора в textarea.
cm.toTextArea()
Удалите редактор и восстановите исходную текстовую область (с помощью текущее содержимое редактора).
cm.getTextArea() → TextAreaElement
Возвращает textarea, на котором был основан экземпляр.
CodeMirror.defaults: object
Объект, содержащий значения по умолчанию для все варианты . Вы можете назначить его свойства для изменения значений по умолчанию (хотя это не повлияет на Редакторы которые уже созданы).
CodeMirror.defineExtension(name: string, value: any)
Если вы хотите определить дополнительные методы в терминах CodeMirror API, можно use defineExtension . Это вызовет данное значение (обычно метод), добавляемое ко всем экземплярам CodeMirror создано с тех пор.
CodeMirror.defineDocExtension(name: string, value: any)
Как defineExtension , но метод будет добавлен в интерфейс для Doc объектов вместо.
CodeMirror.defineOption(name: string, default: any, updateFunc: function)
Аналогично, defineOption может использоваться для определения новых параметров для CodeMirror. The updateFunc will be called with the экземпляр редактора и новое значение при инициализации редактора, и всякий раз, когда опция изменяется через setOption .
CodeMirror.defineInitHook(func: function)
Если вашему расширению просто нужно запустить некоторые код при инициализации экземпляра CodeMirror, use CodeMirror.defineInitHook . Дайте ему функцию как его единственный аргумент, и с тех пор эта функция будет вызываться (с экземпляром в качестве аргумента) всякий раз, когда новый экземпляр CodeMirror инициализируется.
CodeMirror.registerHelper(type: string, name: string, value: helper)
Регистрирует вспомогательное значение с заданным name в заданное пространство имен ( type ). Это используется для определения функциональные возможности, которые могут быть просмотрены в режиме. Создам (если это еще не существует) свойство на CodeMirror объект для данного type , указывающий на объект это сопоставляет имена со значениями. Т. е. после делать CodeMirror.registerHelper("hint", "foo", myFoo) , значение CodeMirror.hint.foo будет указать myFoo .
CodeMirror.Pos(line: integer, ?ch: integer)
Конструктор для {line, ch} объектов, которые используются для представления позиций в документах редактора.
CodeMirror.changeEnd(change: object) → {line, ch}
Служебная функция, вычисляющая конечное положение из изменения (объект с from , to , и text свойства, как передано различные обработчики событий ). Этот возвращенная позиция будет концом измененной диапазон, после применения изменения.

Попробуйте раскомментировать заданный диапазон. Возвращает true , если был найден диапазон комментариев и удалены, false в противном случае.

Переопределение строки комментария свойства режима с пользовательскими строками комментариев.
padding: string
Строка, которая будет вставлена после открытия и до закрытие маркеров комментариев. По умолчанию используется один пробел.

Написание Режимов CodeMirror

Режимы обычно состоят из одного файла JavaScript. Этот файл определяет, в простейшем случае, лексер (токенизатор) для вашего язык-функция, которая принимает символьный поток в качестве входных данных, продвигает его мимо маркера и возвращает стиль для этого маркера. Еще продвинутые режимы также могут обрабатывать отступы для языка.

Сценарий режима должен позвоните CodeMirror.defineMode , чтобы зарегистрироваться CodeMirror. Эта функция принимает два аргумента. Первый должен быть имя режима, для которого следует использовать строчную строку, предпочтительно тот, который также является именем файлов, которые определяют режим (т. е. "xml" определяется в xml.js ). Этот второй аргумент должен быть функцией, заданной CodeMirror объект конфигурации (вещь, переданная CodeMirror функция) и опционный режим объект конфигурации (как в mode опция), возвращает объект mode.

Как правило, следует использовать этот второй аргумент к defineMode как ваша функция объема модуля (режимы ничего не должно просачиваться в глобальные рамки! ), то есть пишите ваши весь режим внутри этой функции.

Основная ответственность скрипта режима- разбор содержимое редактора. В зависимости от языка и количество требуемой функциональности, это может быть сделано в действительно легко или чрезвычайно сложными способами. Некоторые Парсеры могут быть без гражданства, это означает, что они смотрят на один элемент ( маркер ) кода в то время, без памяти о том, что было раньше. Большинство, однако, будет нужно что-то запомнить. Это делается с помощью состояния объект , который является объектом, который всегда передается, когда чтение токена, который может быть изменен с помощью токенизатора.

Режимы, использующие состояние, должны быть определены startState метод для объекта mode. Это функция без аргументов, создающая используемый объект состояния в начале документа.

Наиболее важной частью объекта mode является его token(stream, state) метод. Все режимы должны определите этот метод. Он должен считывать один токен из потока данный в качестве аргумента, при необходимости обновите его состояние и верните строка стиля, или null для маркеров, которые не должны стиль. Для ваших стилей рекомендуется использовать 'стандартные' имена, определенные в темах (без cm- префикс). Если это не удается, это также возможно чтобы придумать свой собственный и написать свой собственный файл темы CSS.

Переданный объект stream для token инкапсуляции строки кода (токены могут never span lines) и наше текущее положение в этой линии. Оно имеет следующий API:

eol() → boolean
Возвращает true, только если поток находится в конце линия.
sol() → boolean
Возвращает true, только если поток находится в начале линия.
peek() → string
Возвращает следующий символ в потоке без продвижения IT. Верну null в конце линия.
next() → string
Возвращает следующий символ в потоке и продвигает его. Также возвращает null , когда нет больше символов доступный.
eat(match: string|regexp|function(char: string) → boolean) → string
match может быть символом, регулярным выражением, или функция, которая принимает символ и возвращает логическое значение. Если следующий символ в потоке "соответствует" заданному аргументу, он потребляется и возвращается. В противном случае, undefined возвращенный.
eatWhile(match: string|regexp|function(char: string) → boolean) → boolean
Многократные вызовы eat с заданным аргументом, пока не выйдет из строя. Возвращает true, если какие-либо символы были съедены.
eatSpace() → boolean
Ярлык для eatWhile сопоставления пробел.
skipToEnd()
Перемещает позицию в конец линии.
skipTo(ch: string) → boolean
Переходит к следующему вхождению данного символа, если найдено на текущей линии (не продвигает поток, если символ не встречается на строке). Возвращает true, если персонаж найден.
match(pattern: string, ?consume: boolean, ?caseFold: boolean) → boolean
match(pattern: regexp, ?consume: boolean) → array<string>
Действуйте как multi-character eat -if consume is true или не дано-или взгляд вперед, который не обновляет поток позиция-если она ложная. pattern может быть либо строка или регулярное выражение, начинающееся с ^ . Когда это строка, caseFold может быть установлено в true сделайте регистр не чувствительным. При успешном сопоставлении регулярное выражение, возвращенное значение будет массивом возвращается match , в случае, если вам нужно извлечь сопоставленные группы.
backUp(n: integer)
Резервное копирование символов потока n . Резервное копирование далее, чем начало текущего токена приведет к перерыв, так что будьте осторожны.
column() → integer
Возвращает столбец (с учетом вкладок), в котором текущий токен запускается.
indentation() → integer
Показывает, как далеко отступ текущей строки, в пространства. Исправляет символы табуляции.
current() → string
Получить строку между началом текущего токена и текущая позиция потока.

По умолчанию пустые строки просто пропускаются, когда маркировка документа. Для языков со значительным пробелом линии, вы можете определить blankLine(state) метод на ваш режим, который будет вызываться всякий раз, когда пустая строка передается over, так что он может обновить состояние анализатора.

Потому что объект состояния мутированы, и CodeMirror необходимо сохранить допустимые версии состояния, чтобы оно могло перезапуск синтаксического анализа в любой строке, копии должны быть сделаны из объектов состояния. По умолчанию используется алгоритм создания нового объекта состояния, который получает все свойства старого объекта. Все свойства которые держат массивы получить копию этих массивов (так как массивы, как правило, использоваться как изменяемые стеки). Когда это не правильно, например поскольку режим изменяет свойства объекта состояния, не являющиеся массивом, a объект mode должен определить a copyState метод, который получает состояние и должен вернуть безопасную копию этого государство.

Если вы хотите, чтобы ваш режим обеспечивал интеллектуальный отступ (через indentLine метод и indentAuto и newlineAndIndent команды, к которым клавиши могут быть bound ), необходимо определить indent(state, textAfter) метод в вашем режиме объект.

Метод отступа должен проверять данный объект состояния, и необязательно textAfter строка, которая содержит текст в строке, отступ и возврат целое число, количество пробелов для отступа. Он должен обычно принимать the indentUnit вариант учитывается. Метод вмятия может возврат CodeMirror.Pass , чтобы указать, что это не смог придумать точного отступа.

Хорошо работать с комментируя аддон , режим может define lineComment (строка, с которой начинается строка комментарий), blockCommentStart , blockCommentEnd (строки, начинающие и заканчивающие комментарии блоков), и blockCommentLead (строка, чтобы положить в начале продолжение строк в комментарии блока). Все это необязательный.

Наконец, режим может определить electricChars свойство, которое должно содержать строку содержит все символы, которые должны вызвать поведение описано для the electricChars выбор.

Итак, подводя итог, режим должен обеспечивать token метод, и он может обеспечьте startState , copyState , и indent методы. Пример тривиального режима, см. режим diff , для больше включенный пример, видит C-как режим .

Иногда полезно для режимов гнездиться -иметь один режим делегировать работу в другой режим. Пример такого рода режим- смешанный режим HTML режим . Для реализации такой вложенности обычно необходимо создайте объекты режима и скопируйте состояния самостоятельно. Создание режима object, there are CodeMirror.getMode(options, parserConfig) , where the first argument is a configuration объект как переданный к функции конструктора режима, и второе аргумент является спецификацией режима, как в mode вариант. Чтобы скопировать a состояние объекта, вызов CodeMirror.copyState(mode, state) , где mode находится режим, который создал данный государство.

В вложенном режиме рекомендуется добавить дополнительные методы, innerMode которые, заданный объект состояния, возвращает {state, mode} объект с внутренним режимом и его состояние для текущей позиции. Они используются утилитой сценарии, такие как тег ближе к получить контекстную информацию. Используйте CodeMirror.innerMode вспомогательная функция, начиная с режима и состояния, рекурсивно спуститесь к внутреннему режиму и состоянию.

Для правильной работы отступа во вложенном синтаксическом анализаторе целесообразно дать startState метод режимов, которые предназначены для вложенного необязательного аргумента, который предоставляет базовый отступ для блока кода. JavaScript и CSS парсер делает это, например, чтобы разрешить JavaScript и CSS код внутри смешанного режима HTML для правильного отступа.

Возможно, и рекомендуется, чтобы связать свой режим, или определенная конфигурация вашего режима, с тип MIME .
